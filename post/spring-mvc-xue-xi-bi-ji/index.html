Spring MVC 学习笔记

2020-12-16

1. Spring MVC 架构图

2. 三层架构

分层是为了解耦，解耦是为了维护方便和分工协作。

2.1 表现层

负责接受 http 请求并完成 http 响应，依赖于业务层。

  • Model
    模型，负责从控制器到视图的数据传递。
  • View
    视图，负责向用户展示数据。
  • Controller
    控制器，负责业务流程的控制。

2.2 业务层

负责业务的逻辑处理，依赖于持久层。

2.3 持久层

负责数据的持久化，即是对数据库的CURD。

3. Spring MVC 请求处理流程

4. 请求参数绑定

4.1 利用 HttpServletRequest 对象

  • request.getParameter("name");

4.2 直接在方法中声明和请求参数名一致的形参

  • 请求参数名和形参名不一致时采用@RequestParam("name")注解映射。
  • Java 对象绑定时，要求请求参数名和属性名一致。

4.3 RESTful 风格的 URL

  • 通过@PathVariable("")注解来获取 URL 中的值。

5. 拦截器的执行流程

  • preHandle() 按配置的顺寻执行，postHandle() 和 afterCompletion() 按配置的反序执行。
