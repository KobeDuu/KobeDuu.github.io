Git 学习笔记

2020-11-25

1. Git 简述

李纳斯·托沃兹（Linus Benedic Torvalds）是 Git 和 Linux 之父

  • 目前最先进的分布式版本控制系统，每个用户的本地仓库都拥有所有的版本信息
  • Git 对待数据并非记录差异，而是直接保存发生改变的文件的快照
  • Git 基于文件内容和目录结构采用 SHA-1 计算校验和来保证数据的完整性

2. Git 架构模型

  • Workspace
    本地工作区，平时存放代码的地方。
  • Index
    本地暂存区，存放临时的改动。
  • Repository
    本地仓库，存放已提交的所有版本信息。
  • Remote
    远程仓库，托管代码的远程服务器。

3. Git 基本指令

  • git init
    初始化本地仓库，生成 .git 文件。
  • git add
    添加文件到暂存区。
  • git commit
    提交暂存区的文件到本地仓库。
  • git clone
    拷贝一份远程仓库，也就是下载一个项目。
  • git status
    查看仓库当前的状态，显示有变更的文件。
  • git diff
    比较文件的不同，即暂存区和工作区的差异。
  • git pull
    下载远程代码并合并。
  • git push
    上传远程代码并合并。
  • git tag
    创建标签
  • git log
    查看当前版本及以前提交的版本信息
  • git reflog
    查看历史版本变化情况

4. 撤销修改

# 撤销工作区的修改，优先退到暂存区的版本，暂存区没有则退到本地仓库的版本
git checkout -- <file>

# 撤销暂存区中关于 file 文件的修改，不会影响工作区
git reset HEAD <file>

# 退回到 commitID 指向的版本
git reset --hard <commitID>

5. 远程托管

# 添加一个名为 url 的远程仓库
git remote add url https://xxx/xxx       
# 推送到远程仓库
git push url master                              
# 查看远程仓库
git remote -v


# 拉取远程仓库与本地合并
git pull --rebase url master 

# 拉取远程仓库代码
git clone url

6. 分支管理
