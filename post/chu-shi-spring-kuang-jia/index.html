初识 Spring 框架

2020-03-21

一. Spring 简介

Spring 是一个轻量级的控制反转（IoC）和面向切面（AOP）的容器框架，使用基本的 JavaBean 代替 EJB，并提供了更多的企业应用功能，目的是解决企业应用开发的复杂性

  • 容器：管理对象的配置和生命周期[1]
  • 框架：将简单的组件配置、组合成为复杂的应用
  • 轻量级：Spring 框架的大小及其所需的处理开销都很小
  • 控制反转：指将对象的创建管理权交给 Spring 容器，不再通过 new 的方式去创建
  • 面向切面：指将动态的代码切入到类的制定方法，指定位置的编程思想，利用 AOP 可以对业务解耦，提高重用性，提升开发效率！

二. Spring 的优点

  1. 支持 AOP
  2. 非侵入式设计
  3. 方便程序测试
  4. 方便解耦、简化开发
  5. 支持声明式事务处理
  6. 方便集成各种优秀框架
  7. 降低 Java EE API 的使用难度

三. Spring 的核心容器

  • BeanFactory：负责配置、创建、管理 Bean[2]，是 Spring 容器最基本的接口；当从容器中拿 Bean 时才会去实例化 Bean，使得启动时占用资源少

  • ApplicationContext：是 BeanFactory 的子接口，不仅包含了BeanFactory 的所有功能，还添加了对国际化、资源访问、事件传播等方面的支持；启动时就实例化所有的 Bean，从容器中拿 Bean 时不再实例化，系统运行速度更快；还可以为 Bean 配置【lazy-init = true】属性实现延迟实例化

四. Spring 的依赖注入

  • 依赖注入（DI）：Spring 容器负责将被依赖对象赋值给调用者的成员变量，相当于为调用者注入了它依赖对象的实例，目的是实现类的解耦。

  • 实现方式：（1）基于构造函数（2）基于设置函数（3）基于自动装配（4）基于注解

五. 控制反转和依赖注入的关系

  • 如果一个类 A 的功能实现需要借助于类 B，那么就称类 B 是类 A 的依赖，如果在类 A 的内部去实例化类 B，那么两者之间会出现较高的耦合，一旦类 B 出现了问题，类 A 也需要进行改造，如果这样的情况较多，每个类之间都有很多依赖，那么就会出现牵一发而动全身的情况，程序会极难维护，并且很容易出现问题。要解决这个问题，就要把 A 类对 B 类的控制权抽离出来，交给一个第三方去做，把控制权反转给第三方，就称作控制反转

  • 控制反转是一种思想，是能够解决问题的一种可能的结果，而依赖注入就是其最典型的实现方法。由第三方（IoC 容器）来控制依赖，把他通过构造函数、属性或者工厂模式等方法，注入到类 A 内，这样就极大程度的对类 A 和类 B 进行了解耦
